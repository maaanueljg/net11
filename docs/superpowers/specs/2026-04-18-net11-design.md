## Resumen

Net11 es una PWA de fantasy fútbol para grupos de amigos. Permite crear equipos con jugadores de una competición elegida (LaLiga, Premier League, Bundesliga, etc.), competir en ligas privadas con un ranking en tiempo real, y en el futuro incorporar puntos reales por jornada gestionados desde un panel de admin.

Cada liga privada está vinculada a una competición concreta. Todos los miembros de esa liga ficharán jugadores únicamente de esa competición, garantizando coherencia y equidad.

**Objetivo de esta fase:** App multijugador funcional, instalable en móvil, con auth real, ligas privadas y ranking en tiempo real.

**Fases futuras:**
- Fase B: Admin panel + puntos reales por jornada
- Fase C: Integración con API de fútbol para puntos automáticos

---

## Stack tecnológico

- **Frontend:** HTML + CSS + Vanilla JS con ES Modules (sin build step)
- **Backend:** Firebase (Firestore + Authentication + Hosting)
- **Auth:** Google OAuth + Email/Contraseña
- **Deploy:** Firebase Hosting → `net11-XXXXX.web.app`
- **PWA:** manifest.json + sw.js (cache-first para assets)

## Autenticación y perfil de usuario

### Proveedores
- Google OAuth (un tap en móvil)
- Email + contraseña (alternativa)
- Sin auth anónimo (eliminado)

### Flujo de primer arranque
1. App abre → comprueba sesión activa
2. Sin sesión → modal de login (no bloquea la UI, pero restringe: no puede guardar equipo ni unirse a ligas)
3. Tras login → si es primera vez, modal de onboarding: campo "Nombre de tu equipo"
4. Sesión persistida por Firebase Auth

### Perfil en Firestore (`users/{uid}`)
```
displayName:   string    ← del proveedor (Google/email)
teamName:      string    ← elegido por el usuario en onboarding
leagues:       string[]  ← códigos de ligas a las que pertenece
updatedAt:     timestamp
```

El equipo, presupuesto y puntos se guardan **por liga** en `users/{uid}/leagueTeams/{leagueCode}` para que un usuario pueda tener un equipo distinto en cada liga (cada una con su propia competición):

```
budget:        number
formation:     string
team:          number[]  ← array de 11 player IDs
totalPts:      number
competition:   string    ← copia de la competición de la liga (para validación)
```

### Tab perfil
El tab "🔥 Firebase" del nav se sustituye por "👤 Perfil":
- Nombre de usuario y nombre del equipo
- Ligas activas (con acceso rápido)
- Botón cerrar sesión

## Sistema de ligas privadas

### Crear liga
1. Usuario pulsa "Nueva Liga" desde tab Perfil o Ranking
2. Introduce nombre de la liga
3. **Elige la competición** — selector visual con las opciones disponibles:
   - 🇪🇸 LaLiga
   - 🏴󠁧󠁢󠁥󠁮󠁧󠁿 Premier League
   - 🇩🇪 Bundesliga
   - 🇮🇹 Serie A
   - 🇫🇷 Ligue 1
   - ⭐ Champions League
4. Se genera código de 6 caracteres (ej: `NET-X7K`)
5. Se muestra código + link compartible (`[hosting-url]/join/NET-X7K`)
6. Botón "Compartir" usa Web Share API (nativo en móvil) o copia al portapapeles
7. El creador queda como admin de la liga

La competición elegida queda bloqueada: no se puede cambiar una vez creada la liga. Todos los miembros verán el mercado y podrán fichar únicamente jugadores de esa competición.

### Unirse a una liga
- Por código: campo de texto en la app
- Por link: la URL `/join/NET-X7K` abre la app directamente en el flujo de join
- Al unirse: usuario añadido a `leagues/{leagueCode}/members[]` y `memberNames`

### Restricciones
- Máximo 20 miembros por liga
- Un usuario puede estar en múltiples ligas
- Solo el admin puede renombrar la liga

### Firestore (`leagues/{leagueCode}`)
```
name:           string
competition:    string     ← 'laliga' | 'premier' | 'bundesliga' | 'seriea' | 'ligue1' | 'champions'
adminUid:       string
members:        string[]   ← uids
memberNames:    map        ← {uid: teamName}
createdAt:      timestamp
currentJornada: number
```

## Ranking multijugador en tiempo real

### Comportamiento
- La pestaña Ranking muestra el ranking de la liga activa del usuario
- Si el usuario está en más de una liga → selector de liga en la parte superior
- Actualización en tiempo real mediante `onSnapshot` de Firestore
- Indicador de conexión activa (punto verde)

### Cálculo de puntos
- `totalPts` = suma de `pts` de los jugadores del equipo (datos estáticos en esta fase)
- Se recalcula en el cliente y se escribe en `users/{uid}/leagueTeams/{leagueCode}.totalPts` al guardar el equipo
- El ranking lee de `users/{uid}/leagueTeams/{leagueCode}` para cada miembro de la liga

### Sin historial de jornadas en esta fase
El historial por jornada se implementa en la Fase B junto con el admin panel.

## Panel de administración (`admin.html`)

### Acceso
- Requiere sesión activa con un uid en lista blanca hardcodeada
- Sin ruta pública — acceso manual por URL

### Funciones
1. **Gestión de jornada:** selector de número de jornada, tabla de jugadores con puntos editables, botón "Publicar Jornada"
2. **Gestión de jugadores:** ver, editar y añadir jugadores (preparando Fase B)

### Publicar jornada
Al publicar, el cliente (navegador del admin) recorre todos los documentos `users/` de Firestore, suma los puntos de los jugadores de cada usuario según `jornadas/{num}.players`, y actualiza `totalPts`. Válido para <100 usuarios sin Cloud Functions.

### Firestore (`jornadas/{jornadaNum}`)
```
published:  boolean
date:       timestamp
players:    map   ← {playerId: pts}
```

## Service Worker y PWA

- `sw.js` con estrategia **cache-first** para HTML, CSS, JS e imágenes
- Fallback offline: sirve la última versión cacheada
- El `manifest.json` ya está configurado → el móvil ofrece "Añadir a pantalla de inicio"
- El link `/join/{code}` está en las rewrites de `firebase.json` para que funcione como SPA

## Reglas de Firestore (ampliadas)

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null && request.auth.uid == userId;

      match /leagueTeams/{leagueCode} {
        allow read: if request.auth != null;
        allow write: if request.auth != null && request.auth.uid == userId;
      }
    }
    match /leagues/{leagueCode} {
      allow read:   if request.auth != null;
      allow create: if request.auth != null;
      allow update: if request.auth != null
                    && request.auth.uid in resource.data.members;
    }
    match /jornadas/{jornadaId} {
      allow read:  if request.auth != null;
      allow write: if request.auth != null
                   && request.auth.uid in ['ADMIN_UID_AQUI'];
    }
  }
}
```

## Datos de jugadores por competición

El fichero `players.js` contendrá jugadores de las 6 competiciones disponibles. El código actual solo tiene LaLiga, Premier y Champions (~50 jugadores). Se ampliarán con datos mock de:

- 🇩🇪 Bundesliga (~15 jugadores)
- 🇮🇹 Serie A (~15 jugadores)
- 🇫🇷 Ligue 1 (~15 jugadores)

Cada jugador tiene el campo `competition` (en lugar de `league`) alineado con el valor usado en `leagues/{code}.competition`.
